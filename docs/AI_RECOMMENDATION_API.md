# การเชื่อมต่อ AI API สำหรับแนะนำรถและการเรียงสินค้า

**อัปเดต:** 2026-02

---

## สรุปคำตอบ

**ได้ครับ** — ระบบสามารถเชื่อมต่อ AI ผ่าน API มาใช้แนะนำรถและให้คำแนะนำการเรียงสินค้า (รวมถึงกรณีที่การจัดเรียงซับซ้อน) ได้ โดย:

1. **แนะนำรถ:** ส่งข้อมูลทริป (สินค้า, ร้าน, น้ำหนัก/ปริมาตรโดยประมาณ) + รายการรถ + ประวัติเมตริกซ์ → AI คืนรถที่เหมาะและเหตุผล
2. **การเรียงสินค้า:** ส่งรายการสินค้า (ขนาด, น้ำหนัก, จำนวน) + ขนาดรถ/พาเลท → AI คืนคำแนะนำข้อความหรือลำดับการจัดเรียง (เช่น ใส่ร้านไหนก่อน, ของหนักไว้ล่าง ฯลฯ)

ยิ่งมี**ข้อมูลเมตริกซ์ที่บันทึกไว้เยอะ** โมเดลยิ่งเรียนรู้จากผลลัพธ์จริง (น้ำหนักจริง, มีปัญหาจัดเรียงหรือไม่) ได้ดี แนะนำได้แม่นขึ้น

---

## โครงสร้างที่แนะนำ

- **ไม่ใส่ API Key ใน Frontend** — เรียก AI ผ่าน **Supabase Edge Function** หรือ Backend ของคุณ โดยเก็บ API Key เป็น secret ฝั่ง server
- **Flow:**
  - แอปเรียก `vehicleRecommendationService.getRecommendations()` (rule-based) เหมือนเดิม
  - (ตัวเลือก) เรียก Edge Function `ai-trip-recommendation` โดยส่งสรุปทริป + รายการรถที่ rule-based คัดมา
  - Edge Function เรียก AI API (เช่น Google Gemini, OpenAI ฯลฯ) แล้วคืน:
    - `suggested_vehicle_id` + `reasoning`
    - `packing_tips` (ข้อความแนะนำการจัดเรียง)
  - แอปแสดงผลจาก rule-based และ/หรือผลจาก AI (รวมถึง packing_tips)

---

## ชื่อ Edge Function

**ชื่อที่ใช้เรียก (invoke):** `ai-trip-recommendation`  
**โฟลเดอร์:** `supabase/functions/ai-trip-recommendation/`

มี Edge Function ตัวอย่างอยู่ที่โฟลเดอร์ดังกล่าว สำหรับ:

- รับ POST body: สรุปทริป (สินค้า, น้ำหนักโดยประมาณ, ร้าน), รายการรถ (id, ความจุ), (ถ้ามี) สรุปประวัติเมตริกซ์
- เรียก AI API ที่คุณเลือก (ในโค้ดตัวอย่างมีโครงสำหรับ Gemini)
- คืน JSON: `{ suggested_vehicle_id?, reasoning?, packing_tips?, error? }`

การตั้งค่า:

- ใส่ API Key ใน Supabase Secrets (เช่น `GEMINI_API_KEY`) ไม่ใส่ในแอป
- ถ้าใช้ AI คนละตัว แก้ใน Edge Function ให้เรียก endpoint และ format ของ API นั้น

---

## ข้อมูลที่ส่งให้ AI (แนะนำ)

เพื่อให้ AI แนะนำรถและให้คำแนะนำการเรียงสินค้าได้ดี ควรส่งอย่างน้อย:

| กลุ่ม | ข้อมูล |
|------|--------|
| ทริป | รายการสินค้า (ชื่อ, จำนวน, น้ำหนัก/หน่วย, ปริมาตร/หน่วย), จำนวนร้าน, วันที่วางแผน |
| โหลดโดยประมาณ | น้ำหนักรวม (กก.), ปริมาตรรวม (ลิตร), จำนวนพาเลทโดยประมาณ (ถ้ามี) |
| รถที่เลือกได้ | รายการ vehicle_id, ความจุ (max_weight_kg, cargo_*), สาขา |
| ประวัติ (ถ้ามี) | จากเมตริกซ์ที่บันทึก: ค่าเฉลี่ยการใช้พื้นที่, น้ำหนักจริง, มีปัญหาจัดเรียงบ่อยหรือไม่ ต่อรถ |

สำหรับ**การเรียงสินค้า** (ซับซ้อน): ส่งขนาดกล่อง/พาเลท, น้ำหนักต่อหน่วย, ข้อจำกัด (ของหนักล่าง, ของแตกง่ายบน ฯลฯ) แล้วให้ AI คืนเป็นข้อความหรือลำดับ (เช่น JSON ของลำดับการโหลด/จัดเรียง).

---

## การนำไปใช้ในแอป

- **Service:** ใน `vehicleRecommendationService` มีเมธอด `getAIRecommendation(params)` แล้ว — ส่ง `trip`, `vehicles`, (ถ้ามี) `historical_context` แล้วได้ `suggested_vehicle_id`, `reasoning`, `packing_tips` คืนมา ถ้าไม่มี API key หรือเรียกไม่สำเร็จจะคืน `null` (ให้ใช้ผล rule-based แทน)
- **แนะนำรถ:** ใช้ rule-based เป็นค่าเริ่มต้น; ถ้าต้องการใช้ AI เรียก `getAIRecommendation()` หลังได้ผล rule-based แล้ว นำ `suggested_vehicle_id` และ `reasoning` มาแสดงหรือปรับลำดับ
- **การเรียงสินค้า:** แสดง `packing_tips` จากผลของ `getAIRecommendation()` ในหน้าสร้างทริป/เลือกรถ (กล่องคำแนะนำใต้รายการสินค้าหรือรถ)

### ตั้งค่า

1. ใส่ API Key ใน Supabase: Dashboard → Project Settings → Edge Functions → Secrets → เพิ่ม `GEMINI_API_KEY`
2. Deploy Edge Function: `supabase functions deploy ai-trip-recommendation`
