# แผนปรับปรุงรายงานการใช้รถละเอียด — โฟกัสความคุ้มค่าและกำไร/ขาดทุน

## 1. วัตถุประสงค์และเป้าหมาย

- **ความตั้งใจ**: ให้รายงานนี้เป็นเครื่องมือวิเคราะห์ สร้างค่าสถิติที่นำไปใช้ตัดสินใจได้
- **เป้าหมายหลัก**:
  - ดู **ความคุ้มค่าของแต่ละทริป** (ต้นทุนต่อทริป, ต้นทุนต่อกม., ต้นทุนต่อชิ้นสินค้า)
  - ดู **ความคุ้มค่าในแต่ละเดือน** (รวมต้นทุน/รายได้ต่อเดือน)
  - **คำนวณกำไร/ขาดทุน** ในช่วงที่เลือก (รายได้ − ต้นทุน) แสดงผลรวมแล้วกำไรหรือขาดทุน

---

## 2. ข้อมูลที่มีในระบบ (ปัจจุบัน)

| แหล่งข้อมูล | ฟิลด์ที่ใช้ได้ | การใช้ |
|-------------|----------------|--------|
| **fuel_records** | vehicle_id, filled_at, liters, price_per_liter, total_cost | ค่าน้ำมันต่อรถต่อช่วงเวลา |
| **commission_logs** | delivery_trip_id, actual_commission | ต้นทุนค่าคอมต่อทริป |
| **trip_logs** | vehicle_id, checkout_time, checkin_time, odometer, distance | จำนวนทริป, ระยะทาง, วันที่ |
| **delivery_trips** | id, planned_date, vehicle_id | ผูกทริปกับวันที่/รถ |
| **vehicle_dashboard** (view) | fuel_cost_last_30_days, maintenance_cost_last_30_days | อ้างอิงต้นทุนรวม 30 วัน (ถ้าใช้) |

**สิ่งที่อาจต้องตรวจเพิ่ม**:  
- รายได้ต่อทริป/ออเดอร์ (orders.total_amount หรือมูลค่าส่งที่ผูกกับ delivery_trip_stores / orders) — ถ้ามีจะคำนวณกำไรจริงได้

---

## 3. สถิติที่เสนอ (นำไปวิเคราะห์ได้)

### 3.1 ระดับช่วงที่เลือก (รถคันเดียว + date range)

- **ต้นทุนรวม**
  - ค่าน้ำมันรวม (จาก fuel_records ในช่วง + vehicle_id)
  - ค่าคอมรวม (จาก commission_logs ของ delivery_trip ที่อยู่ในช่วง)
  - (ถ้ามี) ค่าซ่อมที่ผูกกับรถ/ช่วง — ใช้ maintenance หรือ tickets ถ้าผูกกับช่วงเวลาได้
- **รายได้รวม** (ถ้ามีข้อมูล order/มูลค่าส่ง)
  - มูลค่าส่งหรือยอดออเดอร์ที่เชื่อมกับทริปในช่วง
- **กำไร/ขาดทุน**
  - กำไร = รายได้รวม − ต้นทุนรวม
  - แสดงเป็นตัวเลขและเป็น % ของรายได้ (ถ้ามีรายได้)
- **ตัวชี้ความคุ้มค่า (ไม่มีรายได้ก็ใช้ได้)**
  - ต้นทุนต่อทริป (บาท/ทริป)
  - ต้นทุนต่อกม. (บาท/กม.)
  - ต้นทุนต่อชิ้นสินค้าส่ง (บาท/ชิ้น) — จากจำนวนชิ้นที่ส่งในทริป

### 3.2 ระดับรายเดือน (รถคันเดียว)

- แยกตามเดือน: จำนวนทริป, ระยะทาง, ค่าน้ำมัน, ค่าคอม, (ค่าซ่อม), (รายได้), กำไร/ขาดทุน
- สรุปเป็นตารางหรือกราฟแนวเส้น/แท่ง: ต้นทุน vs รายได้ vs กำไร ต่อเดือน

### 3.3 ระดับต่อทริป (ถ้าต้องการเจาะลึก)

- ต่อทริป: วันที่, เลขทริป, ระยะทาง, ค่าน้ำมัน(ประมาณ), ค่าคอม, (รายได้), กำไร/ขาดทุนต่อทริป
- ใช้วิเคราะห์ว่าทริปไหนคุ้ม/ไม่คุ้ม

---

## 4. การคำนวณที่ต้องทำ

1. **ค่าน้ำมันในช่วง**
   - Query `fuel_records` ที่ vehicle_id = รถที่เลือก และ filled_at อยู่ใน startDate–endDate
   - Sum(total_cost) หรือ sum(liters * price_per_liter) ถ้า total_cost เป็น null
2. **ค่าคอมในช่วง**
   - จาก delivery_trip ที่ vehicle_id + planned_date อยู่ในช่วง → หา delivery_trip_id ทั้งหมด
   - Sum(actual_commission) จาก commission_logs ที่ delivery_trip_id ในเซตนี้
3. **รายได้ในช่วง** (ถ้ามี)
   - ผูก orders / delivery_trip_stores กับ delivery_trip และวันที่
   - Sum(mูลค่าส่งหรือ order total) ตามช่วง
4. **กำไร = รายได้รวม − ต้นทุนรวม** (ต้นทุน = น้ำมัน + ค่าคอม + อื่นๆ ถ้ามี)

---

## 5. UI/UX ที่เสนอ

### 5.1 โครงหน้ารายงาน (จัดเรียงใหม่ให้โฟกัสสถิติ)

1. **ตัวกรอง** (รถ + ช่วงวันที่) — เหมือนเดิม, อาจเพิ่มตัวเลือก “ดูแบบรายเดือน”
2. **การ์ดสรุปตัวเลขหลัก (ด้านบน)**
   - ทริปทั้งหมด, ระยะทางรวม, วันที่มีใช้รถ (เหมือนเดิม)
   - **เพิ่ม**: ค่าน้ำมันรวม (บาท), ค่าคอมรวม (บาท), **ต้นทุนรวม (บาท)**
   - **เพิ่ม**: รายได้รวม (บาท) — ถ้ามีข้อมูล
   - **เพิ่ม**: **กำไร/ขาดทุน (บาท และ %)** พร้อมสีเขียว/แดง
3. **ส่วนความคุ้มค่า (ตัวชี้)**
   - ต้นทุนต่อทริป, ต้นทุนต่อกม., ต้นทุนต่อชิ้น (ถ้ามีจำนวนชิ้น)
4. **กราฟ**
   - กราฟสินค้า (คงเดิม)
   - **เพิ่ม**: กราฟแนวเส้นหรือแท่ง “ต้นทุน vs รายได้ vs กำไร ต่อเดือน” (เมื่อเลือกช่วงยาวหรือโหมดรายเดือน)
5. **ตารางสรุปรายวัน** (คงเดิม + คอลัมน์เสริมถ้าต้องการ)
   - อาจเพิ่มคอลัมน์ “ค่าน้ำมัน(ประมาณ)” หรือ “ค่าคอม” ต่อวัน ถ้าคำนวณได้
6. **ตารางสรุปรายเดือน** (ใหม่)
   - หนึ่งแถวต่อเดือน: เดือน, จำนวนทริป, ระยะทาง, ค่าน้ำมัน, ค่าคอม, (รายได้), กำไร/ขาดทุน
7. **ปุ่ม Export**
   - Export สถิติ (CSV/Excel) สำหรับนำไปวิเคราะห์ต่อ

### 5.2 การจัดเรียงและโหมดดู

- **โหมดช่วงวันที่ (ปัจจุบัน)**: แสดงสรุปรวมช่วง + รายวัน (แบ่งหน้า)
- **โหมดรายเดือน**: สลับให้แสดง “สรุปรายเดือน” เป็นตารางหลัก + กราฟรายเดือน
- ตัวกรอง “ช่วงวันที่” ยังใช้ได้กับทั้งสองโหมด (กำหนดขอบเขตข้อมูล)

## 6. ขั้นตอนการพัฒนา (Phase)

### Phase 1 — ต้นทุนและตัวชี้ความคุ้มค่า (ไม่พึ่งรายได้)

- [ ] Service: ดึงค่าน้ำมันรวมของรถในช่วง (fuel_records by vehicle_id + filled_at)
- [ ] Service: ดึงค่าคอมรวมของทริปที่รถคันนี้ในช่วง (commission_logs join delivery_trips)
- [ ] Hook/Service: คืนค่า cost summary (fuel, commission, total) ต่อช่วงที่เลือก
- [ ] UI: เพิ่มการ์ด “ต้นทุนรวม (บาท)” และแยกค่าน้ำมัน/ค่าคอม
- [ ] UI: เพิ่มตัวชี้ “ต้นทุนต่อทริป”, “ต้นทุนต่อกม.”, “ต้นทุนต่อชิ้น”
- [ ] (Optional) ตารางรายวัน: คอลัมน์ค่าน้ำมัน/ค่าคอมต่อวัน ถ้าคำนวณได้

### Phase 2 — สรุปรายเดือนและกราฟ

- [ ] Service: สรุปต้นทุน (และรายได้ถ้ามี) แยกตามเดือน สำหรับรถที่เลือก + ช่วงวันที่
- [ ] UI: ตาราง “สรุปรายเดือน” (เดือน, ทริป, ระยะทาง, ค่าน้ำมัน, ค่าคอม, ต้นทุนรวม)
- [ ] UI: กราฟแนวเส้น/แท่ง ต้นทุน (และรายได้) ต่อเดือน
- [ ] ตัวเลือก “ดูแบบรายเดือน” หรือสลับมุมมองช่วง/รายเดือน

### Phase 3 — รายได้และกำไร/ขาดทุน

- [ ] ตรวจสอบ schema: orders / delivery_trip_stores มีมูลค่า/รายได้ผูกกับทริปหรือไม่
- [ ] Service: คำนวณรายได้รวมต่อช่วง (และต่อเดือน ถ้า Phase 2 ทำแล้ว)
- [ ] UI: การ์ด “รายได้รวม”, “กำไร/ขาดทุน (บาท และ %)” พร้อมสีสถานะ
- [ ] UI: คอลัมน์รายได้/กำไรในตารางรายเดือน
- [ ] กราฟ: เพิ่มเส้นหรือกลุ่มรายได้และกำไรในกราฟรายเดือน

### Phase 4 — Export และการนำไปวิเคราะห์

- [ ] ปุ่ม Export (CSV/Excel): สรุปช่วง + สรุปรายวัน + สรุปรายเดือน (ตัวเลขที่แสดงในหน้านั้น)
- [ ] (Optional) ฟิลเตอร์เพิ่ม เช่น สาขา, ประเภทรถ

---

## 7. หมายเหตุการออกแบบ

- **ความคุ้มค่า**: เน้นตัวชี้ที่ใช้ได้แม้ยังไม่มี “รายได้” ในระบบ (ต้นทุนต่อทริป/กม./ชิ้น)
- **กำไร/ขาดทุน**: จะแม่นยำเมื่อมีข้อมูลรายได้ที่ผูกกับทริป/ออเดอร์ — ถ้ายังไม่มี ควรระบุใน UI ว่า “แสดงเฉพาะต้นทุน รายได้และกำไรจะแสดงเมื่อมีข้อมูล”
- **ประสิทธิภาพ**: ดึง fuel + commission แยก query แล้วรวมฝั่ง frontend หรือสร้าง view/function ใน DB ถ้าช่วงยาวหรือหลายรถ
- **RLS**: ทุก query ต้องอยู่ภายใต้ RLS ของ Supabase (ใช้ anon key / role ปัจจุบัน)

---

## 8. สรุป

| เป้าหมาย | วิธีดำเนินการ |
|----------|----------------|
| ความคุ้มค่าของแต่ละทริป | ตัวชี้ต้นทุนต่อทริป/กม./ชิ้น + (Phase 3) กำไรต่อทริปถ้ามีรายได้ |
| ความคุ้มค่าในแต่ละเดือน | ตารางและกราฟสรุปรายเดือน (Phase 2) |
| กำไร/ขาดทุนรวม | การ์ดและตารางรายเดือนแสดง กำไร = รายได้ − ต้นทุน (Phase 3 หลังมีรายได้) |
| นำไปวิเคราะห์ต่อ | Export CSV/Excel (Phase 4) |

ถ้าต้องการเริ่มจาก Phase 1 สามารถทำได้โดยไม่ต้องมีข้อมูลรายได้ในระบบ และค่อยเพิ่มรายได้/กำไรใน Phase 3 เมื่อข้อมูลพร้อม
