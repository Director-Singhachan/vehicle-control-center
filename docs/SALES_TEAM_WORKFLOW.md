# 👥 Sales Team Workflow & Base of Operations

## 📍 ฐานบัญชาการของฝ่ายขาย

### 🎯 แอพหลัก: **Web App (Vehicle Control Center)**

ฝ่ายขายจะใช้ **Web App (React)** เป็นฐานบัญชาการหลักสำหรับ:

#### ✅ งานหลักที่ทำใน Web App
1. **รับออเดอร์จากลูกค้า**
   - ดูออเดอร์ใหม่ที่ลูกค้าสั่ง
   - รับออเดอร์เข้าสู่ระบบ

2. **ส่งออเดอร์ให้คลังและจัดส่ง**
   - ส่งข้อมูลออเดอร์ให้คลัง
   - อัปเดตสถานะเป็น 'sent_to_warehouse'

3. **ดูคิวที่คลังจัดให้**
   - ดู delivery trips ที่คลังจัดคิวแล้ว
   - ดูลำดับร้านที่จะส่ง (sequence_order)
   - ดูว่าควรออกบิลร้านไหนก่อน

4. **ออกบิลตามลำดับคิว**
   - ออกบิลตามลำดับที่คลังกำหนด
   - อัปเดตสถานะเป็น 'billed'

5. **จัดการลูกค้า**
   - ดูข้อมูลร้านค้า/ลูกค้า
   - เพิ่ม/แก้ไขข้อมูลลูกค้า
   - ดูประวัติการสั่งซื้อ

6. **รายงานและสถิติ**
   - ดูยอดขาย
   - ดูสถิติออเดอร์

---

### 🌐 Web App: **สำหรับทั้งฝ่ายขายและคลังและจัดส่ง**

**หมายเหตุ:** Web App เป็นเครื่องมือของ **ทั้งฝ่ายขายและคลังและจัดส่ง**

- **ฝ่ายขาย**: ใช้ Web App สำหรับรับออเดอร์, ส่งให้คลัง, ดูคิว, ออกบิล
- **คลังและจัดส่ง**: ใช้ Web App สำหรับจัดคิว, จัดรถ, ติดตามการส่ง

#### ✅ งานที่คลังและจัดส่งทำใน Web App
1. **ดูออเดอร์ที่ขายส่งมา**
   - ดูออเดอร์ status = 'sent_to_warehouse'
   - กรองตามวันที่

2. **จัดคิวส่ง**
   - เลือกร้านที่จะส่ง
   - กำหนดลำดับร้าน (sequence_order)
   - สร้าง delivery trip

3. **ส่งคิวให้ขาย**
   - อัปเดต status = 'queue_ready'
   - แจ้งเตือนให้ขาย

4. **จัดการการส่ง**
   - จัดรถ
   - กำหนดพนักงานขับ
   - ติดตามการส่งสินค้า

---

## 🌐 Web App - Sales Features

### Dashboard Screen
- จำนวนออเดอร์ใหม่
- ออเดอร์ที่รอส่งให้คลัง
- ออเดอร์ที่คลังจัดคิวแล้ว (พร้อมออกบิล)
- ยอดขายวันนี้/เดือนนี้

### Orders Management
- **New Orders**: ออเดอร์ใหม่ที่ลูกค้าสั่ง
- **Sent to Warehouse**: ออเดอร์ที่ส่งให้คลังแล้ว
- **Queue Ready**: ออเดอร์ที่คลังจัดคิวแล้ว (พร้อมออกบิล)
- **Billed**: ออเดอร์ที่ออกบิลแล้ว
- **All Orders**: ออเดอร์ทั้งหมด
- **Search & Filter**: ค้นหาออเดอร์ตามวันที่, สถานะ, ลูกค้า

### Order Detail Screen
- ดูรายละเอียดออเดอร์
- ดูรายการสินค้า
- ปุ่ม "ส่งให้คลัง" (ถ้ายังไม่ได้ส่ง)
- แก้ไขออเดอร์ (ถ้าจำเป็น)

### Queue View (ดูคิวที่คลังจัด)
- ดู delivery trips ที่พร้อมให้ออกบิล
- ดูลำดับร้านที่จะส่ง (sequence_order)
- เรียงตามลำดับที่คลังกำหนด
- ปุ่ม "ออกบิล" สำหรับแต่ละร้าน

### Customer Management
- ดูรายชื่อลูกค้า
- เพิ่ม/แก้ไขข้อมูลลูกค้า
- ดูประวัติการสั่งซื้อ

### Reports & Analytics
- ยอดขายรายวัน/รายเดือน
- สถิติออเดอร์
- รายงานการส่งสินค้า
- Top customers

## 🌐 Web App - Sales Features (เสริม)

### Delivery Trip Management
- ดู delivery trips ทั้งหมด
- จัดคิวรถตามออเดอร์
- กำหนดพนักงานขับรถ
- ติดตามสถานะการส่ง

### Vehicle Status
- ดูสถานะรถ
- ตรวจสอบรถว่าง/ไม่ว่าง
- ดูประวัติการใช้งาน

### Reports
- รายงานการส่งสินค้า
- สถิติการใช้รถ
- ต้นทุนการส่งสินค้า

---

## 🔐 Access Control

### Roles สำหรับฝ่ายขาย

1. **`sales`** - พนักงานขาย
   - Mobile App: Full access (จัดการออเดอร์, ลูกค้า, สร้าง delivery trips)
   - Web App: Read-only (ดู delivery trips, ติดตามการส่ง)

2. **`manager`** - ผู้จัดการฝ่ายขาย
   - Mobile App: Full access + Reports
   - Web App: Full access (จัดการ delivery trips, ดูรายงาน)

3. **`admin`** - ผู้ดูแลระบบ
   - Mobile App: Full access
   - Web App: Full access

## 📊 Database Tables ที่ฝ่ายขายใช้

### Mobile App (หลัก)
- `orders` - ออเดอร์
- `order_items` - รายการสินค้าในออเดอร์
- `order_status_history` - ประวัติการเปลี่ยนสถานะ
- `stores` - ร้านค้า/ลูกค้า
- `products` - สินค้า
- `profiles` - ข้อมูลผู้ใช้

### Web App (เสริม)
- `delivery_trips` - ทริปส่งสินค้า
- `delivery_trip_stores` - ร้านค้าในทริป
- `delivery_trip_items` - รายการสินค้าในทริป
- `vehicles` - รถ
- `trip_logs` - บันทึกการใช้งานรถ

## 🚀 การเริ่มใช้งาน

### สำหรับ Sales Team

1. **ติดตั้ง Mobile App**
   - ดาวน์โหลดจาก App Store / Play Store
   - Login ด้วย email/password
   - ระบบจะตรวจสอบ role = 'sales' หรือ 'manager'

   - Login ด้วย account เดียวกัน
   - ใช้สำหรับติดตามการส่งสินค้า

3. **เริ่มทำงาน**
   - ดู Dashboard → ดูออเดอร์ใหม่
   - อนุมัติออเดอร์ → สร้าง delivery trip
   - ติดตามการส่งสินค้า

## 🎯 Best Practices

1. **ใช้ Mobile App เป็นหลัก** - สำหรับงานประจำวัน
2. **ใช้ Web App เมื่อต้องการ** - ดูรายละเอียดการส่งสินค้า, จัดคิวรถ
3. **ตรวจสอบออเดอร์เป็นประจำ** - อย่าปล่อยให้ออเดอร์ค้างนาน
4. **สร้าง Delivery Trip ทันที** - หลังจากอนุมัติออเดอร์แล้ว
5. **ติดตามการส่งสินค้า** - ตรวจสอบสถานะเป็นระยะ
