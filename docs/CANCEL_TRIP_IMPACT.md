# ผลกระทบของการยกเลิกทริป (Cancel Trip Impact)

## สรุป
เมื่อยกเลิกทริปส่งสินค้า (delivery_trips) หรือทริปการใช้งานรถ (trip_logs) ระบบจะเปลี่ยน status เป็น `'cancelled'` **ไม่ใช่การลบข้อมูล** ดังนั้นทริปที่ยกเลิกจะยังอยู่ในระบบ

## ผลกระทบต่อการคำนวณผล

### 1. รายงาน (Reports)
- **ทริปที่ยกเลิกจะไม่ถูกนับ** ในรายงานต่างๆ เพราะ:
  - Reports จะกรองเฉพาะ status = `'completed'` หรือ `'in_progress'` เท่านั้น
  - ทริปที่ status = `'cancelled'` จะถูกกรองออกอัตโนมัติ
- **ผลกระทบ**: ไม่มีผลกระทบต่อการคำนวณผล เพราะทริปที่ยกเลิกไม่ควรถูกนับอยู่แล้ว

### 2. Sequence Order
- **sequence_order จะไม่ถูกคำนวณใหม่** เมื่อยกเลิกทริป
- **เหตุผล**: 
  - ถ้าคำนวณใหม่จะทำให้ลำดับสับสน (เช่น ถ้าทริปที่ 8 ถูกยกเลิก แล้วทริปที่ 9, 10 จะกลายเป็น 8, 9 ซึ่งไม่ถูกต้อง)
  - sequence_order ควรคงที่ตามลำดับที่สร้างทริป
- **ผลกระทบ**: 
  - อาจมีช่องว่างใน sequence_order (เช่น 1, 2, 3, 5, 6, 7, 8, 9, 10 ถ้าทริปที่ 4 ถูกยกเลิก)
  - แต่ไม่กระทบการทำงานของระบบ เพราะการเรียงลำดับจะใช้ sequence_order + planned_date + created_at

### 3. การแสดงผลใน UI
- **DeliveryTripListView**: 
  - ทริปที่ยกเลิกจะไม่แสดงในรายการปกติ (filter default ไม่รวม 'cancelled')
  - สามารถดูทริปที่ยกเลิกได้โดยเลือก filter "ยกเลิก" หรือ "ทั้งหมด"
- **TripLogListView**: 
  - ทริปที่ยกเลิกจะไม่แสดงในรายการปกติ
  - สามารถดูได้โดยเลือก filter "ยกเลิกแล้ว"

### 4. การเชื่อมโยงกับ trip_logs
- ถ้า trip_log มี `delivery_trip_id` ที่ชี้ไปยัง delivery_trip ที่ถูกยกเลิก:
  - trip_log จะยังคงมี `delivery_trip_id` อยู่ (ไม่ถูกลบ)
  - แต่ delivery_trip.status จะเป็น `'cancelled'`
  - เมื่อ query trip_logs จะยังเห็น delivery_trip ที่ถูกยกเลิกได้

## เงื่อนไขในการยกเลิกทริป

### Delivery Trips
- **ยกเลิกได้**: status = `'planned'` หรือ `'in_progress'`
- **ยกเลิกไม่ได้**: status = `'completed'` หรือ `'cancelled'`
- **สิทธิ์**: 
  - Admin, Manager, Inspector: ยกเลิกได้ทุกทริป
  - Driver: ยกเลิกได้เฉพาะทริปของตัวเอง (driver_id = auth.uid())

### Trip Logs
- **ยกเลิกได้**: status = `'checked_out'` (ออกไปแล้วแต่ยังไม่กลับ)
- **ยกเลิกไม่ได้**: status = `'checked_in'` หรือ `'cancelled'`
- **สิทธิ์**: 
  - Admin, Manager, Inspector: ยกเลิกได้ทุกทริป
  - Driver: ยกเลิกได้เฉพาะทริปของตัวเอง (driver_id = auth.uid())

## สรุป
- ✅ **การคำนวณผลไม่ได้รับผลกระทบ** เพราะทริปที่ยกเลิกจะถูกกรองออกอัตโนมัติ
- ✅ **sequence_order ไม่ถูกคำนวณใหม่** (ตามที่ออกแบบไว้)
- ✅ **ข้อมูลยังคงอยู่ในระบบ** เพื่อ audit trail
- ⚠️ **ต้องมีสิทธิ์ที่เหมาะสม** ในการยกเลิกทริป
